## Key findings from sdram_test_01

### M0b model change (critical for timing)

sdram_test_01 sdram_model_pro.sv changed from `cas_pipe[CL+1]` to `cas_pipe[CL]` (3 stages).
Result: DQ NBA fires at T_sdram_rd + CL = T_sdram_rd + 3 clk_sh cycles.

Where T_sdram_rd = 1 clk_sh cycle after PHY CMD FF captures READ command.

### RSHIFT=-1 failure (E007)

READ_IDX = CAS_LATENCY + RSHIFT = 3 + (-1) = 2.
PHY captures at T_cmd+2 clk_sh cycles from read_det entering pipeline.
Model DQ NBA at T_cmd+3.
Race: PHY captures PRE-NBA DQ (Hi-Z). Result: dq_i = Hi-Z.

### RSHIFT=0 failure (E008)

READ_IDX = 3. PHY captures at T_cmd+3.
Single read tests PASS (PHY/scheduler).
AXI 2-read path FAILS with 0xzzzz1234.
Root cause: unknown extra cycle in AXI 2-read path vs single read.

## Status at handover

sdram_test_02 goal: identify the correct RSHIFT and understand the extra cycle in 2-read path.

---

## sdram_test_02 resolution (2026-06-21)

sdram_test_02 resolved the isolated PHY/word-assembly question.

- Correct RSHIFT confirmed as 0 (minimum safe) with CMD_REG_ON_CLK_SH=1 and CL=3.
- PHY single read, back-to-back reads, and 32-bit word assembly all PASS in isolation.
- The AXI 2-read failure from sdram_test_01 (E008: 0xzzzz1234) is NOT caused by the raw PHY or assembler.

The remaining bug from sdram_test_01 is likely above the raw PHY+assembler layer.
Next investigation should target native read/write engine and scheduler integration before returning to AXI.
