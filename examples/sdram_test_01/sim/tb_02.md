Nižšie máš **profesionálnejší SDR SDRAM behavioral model (`sdram_model_pro.sv`)**, ktorý je stále relatívne kompaktný, ale už implementuje základné **timing obmedzenia a bank/row logiku** podobne ako referenčné modely výrobcov pamätí (napr. Micron Technology alebo Samsung Electronics).

Podporuje:

* 4 banky
* ACTIVE / PRECHARGE
* READ / WRITE
* CAS latency pipeline
* tRCD
* tRP
* tRAS
* row tracking
* jednoduchý burst (BL=1)

Nie je to plný JEDEC model (~3000 LOC), ale **na testovanie controlleru je to ideálne**.

---

# `rtl/sdram_model_pro.sv`

```systemverilog
/**
 * @file sdram_model_pro.sv
 * @brief Professional SDR SDRAM behavioral model (simplified)
 *
 * Features:
 *  - 4 banks
 *  - ACTIVE / PRECHARGE
 *  - READ / WRITE
 *  - CAS latency pipeline
 *  - tRCD, tRP, tRAS enforcement
 *
 * Not full JEDEC timing model, but close enough
 * for controller verification.
 */

`timescale 1ns/1ps
`default_nettype none

module sdram_model_pro #(

  parameter DATA_WIDTH = 16,
  parameter ROW_WIDTH  = 13,
  parameter COL_WIDTH  = 10,
  parameter BANKS      = 4,

  // timing in cycles
  parameter tRCD = 2,
  parameter tRP  = 2,
  parameter tRAS = 4,
  parameter CL   = 3

)(

  input logic clk,

  input logic [ROW_WIDTH-1:0] addr,
  input logic [1:0] ba,

  input logic ras_n,
  input logic cas_n,
  input logic we_n,
  input logic cs_n,

  inout wire [DATA_WIDTH-1:0] dq

);

  // ==========================================================
  // MEMORY ARRAY
  // ==========================================================

  localparam MEM_DEPTH = 1<<20;

  logic [DATA_WIDTH-1:0] mem [0:MEM_DEPTH-1];

  // ==========================================================
  // BANK STATE
  // ==========================================================

  typedef struct packed {

    logic open;
    logic [ROW_WIDTH-1:0] row;

    int rcd_timer;
    int ras_timer;
    int rp_timer;

  } bank_state_t;

  bank_state_t bank [BANKS];

  // ==========================================================
  // DQ BUS
  // ==========================================================

  logic [DATA_WIDTH-1:0] dq_out;
  logic dq_drive;

  assign dq = dq_drive ? dq_out : 'z;

  wire [DATA_WIDTH-1:0] dq_in = dq;

  // ==========================================================
  // CAS PIPELINE
  // ==========================================================

  typedef struct packed {
    logic valid;
    logic [DATA_WIDTH-1:0] data;
  } cas_pipe_t;

  cas_pipe_t cas_pipe [CL];

  // ==========================================================
  // COMMAND DECODE
  // ==========================================================

  wire cmd_active   = (!cs_n && !ras_n &&  cas_n &&  we_n);
  wire cmd_pre      = (!cs_n && !ras_n &&  cas_n && !we_n);
  wire cmd_read     = (!cs_n &&  ras_n && !cas_n &&  we_n);
  wire cmd_write    = (!cs_n &&  ras_n && !cas_n && !we_n);

  // ==========================================================
  // TIMING COUNTERS
  // ==========================================================

  integer i;

  always_ff @(posedge clk) begin

    for(i=0;i<BANKS;i++) begin

      if(bank[i].rcd_timer > 0)
        bank[i].rcd_timer--;

      if(bank[i].ras_timer > 0)
        bank[i].ras_timer--;

      if(bank[i].rp_timer > 0)
        bank[i].rp_timer--;

    end

  end

  // ==========================================================
  // COMMAND PROCESSING
  // ==========================================================

  always_ff @(posedge clk) begin

    dq_drive <= 0;

    // shift CAS pipeline
    for(i=CL-1;i>0;i--)
      cas_pipe[i] <= cas_pipe[i-1];

    cas_pipe[0].valid <= 0;

    if(cas_pipe[CL-1].valid) begin
      dq_out   <= cas_pipe[CL-1].data;
      dq_drive <= 1;
    end

    if(!cs_n) begin

      // -------------------------------------------------------
      // ACTIVE
      // -------------------------------------------------------

      if(cmd_active) begin

        if(bank[ba].open) begin
          $error("ACTIVE issued to open bank");
        end

        if(bank[ba].rp_timer > 0) begin
          $error("tRP violation");
        end

        bank[ba].open <= 1;
        bank[ba].row  <= addr;

        bank[ba].rcd_timer <= tRCD;
        bank[ba].ras_timer <= tRAS;

      end

      // -------------------------------------------------------
      // PRECHARGE
      // -------------------------------------------------------

      if(cmd_pre) begin

        if(!bank[ba].open) begin
          $error("PRECHARGE on closed bank");
        end

        if(bank[ba].ras_timer > 0) begin
          $error("tRAS violation");
        end

        bank[ba].open <= 0;
        bank[ba].rp_timer <= tRP;

      end

      // -------------------------------------------------------
      // READ
      // -------------------------------------------------------

      if(cmd_read) begin

        if(!bank[ba].open)
          $error("READ on closed bank");

        if(bank[ba].rcd_timer > 0)
          $error("tRCD violation");

        int index;

        index = {ba,bank[ba].row,addr[COL_WIDTH-1:0]};

        cas_pipe[0].valid <= 1;
        cas_pipe[0].data  <= mem[index];

      end

      // -------------------------------------------------------
      // WRITE
      // -------------------------------------------------------

      if(cmd_write) begin

        if(!bank[ba].open)
          $error("WRITE on closed bank");

        if(bank[ba].rcd_timer > 0)
          $error("tRCD violation");

        int index;

        index = {ba,bank[ba].row,addr[COL_WIDTH-1:0]};

        mem[index] <= dq_in;

      end

    end

  end

endmodule
```

# Čo tento model umožní debugovať

Ak controller spraví chybu, uvidíš napr.:

```
tRCD violation
ACTIVE issued to open bank
READ on closed bank
tRAS violation
```

To je **rovnaký typ verifikácie**, aký používajú výrobcovia SDRAM.

---

✅ Ak chceš, môžem ti ešte pripraviť **extrémne užitočný nástroj pre vývoj SDRAM controllerov**:

**SDRAM command trace analyzer**, ktorý počas simulácie vypisuje sekvencie ako:

```
ACT B0 ROW=123
READ B0 COL=45
READ B0 COL=46
PRE B0
```

To dramaticky zrýchľuje debug controllerov.
