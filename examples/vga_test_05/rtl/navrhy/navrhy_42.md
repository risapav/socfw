## Čo je aktuálne v `soc_top.sv`

Tento `soc_top.sv` je opäť test **2B GCP-only**, nie fázový T1/T2/T3 test:

```systemverilog
.DEBUG_ISLAND_PHASES(0),
.ENABLE_ACR_PACKET(0),
.ENABLE_AUDIO(0),
.ENABLE_AUDIO_INFOFRAME(0),
.ENABLE_AUDIO_SAMPLE(0),
.ENABLE_AVI_PACKET(0),
.ENABLE_DATA_ISLAND(1),
.ENABLE_GCP_PACKET(1),
.GCP_FRAME_PERIOD(1),
.VBLANK_ONLY(1)
```

Čiže:

```text
DATA=1
AUDIO=0
GCP=1
AVI=0
VBLANK_ONLY=1
DEBUG_ISLAND_PHASES=0
```

To je normálny plný GCP data-island packet vo vblanku. Keďže 2B na FPGA zlyháva, tento bitstream podľa očakávania stále testuje zlyhávajúci prípad.

## Čo refaktor obsahuje správne

### 1. `DEBUG_ISLAND_PHASES` je v scheduleri zapracovaný

V `hdmi_period_scheduler.sv` vidím:

```systemverilog
parameter int DEBUG_ISLAND_PHASES = 0
```

a fázy:

```text
1 = T1: DATA_PREAMBLE only
2 = T2: DATA_PREAMBLE + guard bands, bez payloadu
3 = T3: DATA_PREAMBLE + guard bands + 1 payload symbol
0 = normálny full packet
```

To je presne správna diagnostická cesta.

---

### 2. `VBLANK_ONLY` je zapracované

Start data islandu má podmienku:

```systemverilog
(!VBLANK_ONLY || vblank_i)
```

Čiže pri `VBLANK_ONLY=1` sa pakety vkladajú len počas vertical blankingu.

---

### 3. Lookahead `packet_pop_o` je stále prítomný

V `ST_DATA_GUARD_LEAD`:

```systemverilog
packet_pop_o = 1'b1;
```

a v payload stave:

```systemverilog
packet_pop_o = (DEBUG_ISLAND_PHASES == 0) ? (sym_cnt > 6'd1) : 1'b0;
```

To znamená, že starý problém typu:

```text
symbol0, symbol0, symbol1, ..., symbol30
```

by mal byť odstránený.

---

### 4. TERC4 encoder je 2-cyklový

V `terc4_encoder.sv` je:

```systemverilog
nibble_i → nibble_r → lut → tmds_o
```

Teda TERC4 pipeline sedí s video/control cestou.

---

### 5. GCP builder generuje all-zero GCP

`gcp_packet_builder.sv` pri aktuálnom zapojení v core dáva:

```text
HB = 00 00 00
PB = all zero
```

lebo `avmute_i=0` a `clear_avmute_i=0`.

Takže 2B nezlyháva kvôli AVMUTE alebo color-depth bitom.

---

## Jeden problém v komentári

V `data_island_formatter.sv` je stále zastaraný usage komentár:

```systemverilog
Assert advance_i each clock during the 32-cycle payload phase
```

To už nie je presné, pretože scheduler teraz používa lookahead a v normal režime neposiela 32 payload popov, ale skorší advance + následné obmedzené posuny.

Odporúčam komentár zmeniť na niečo ako:

```systemverilog
// advance_i is generated by hdmi_period_scheduler as a pipeline-lookahead
// pulse. It is not one pulse per visible DATA_PAYLOAD cycle.
// The scheduler advances the formatter early enough so that after the
// 2-cycle TERC4 encoder and registered channel mux, ch*_o emits payload
// symbols 0..31 exactly once during DATA_PAYLOAD.
```

Funkčne to nie je bug, ale dokumentačne je to nebezpečné.

## Najbližší test: T2

Nastav:

```systemverilog
.DEBUG_ISLAND_PHASES(2),
.ENABLE_DATA_ISLAND(1),
.ENABLE_AUDIO(0),
.ENABLE_GCP_PACKET(1),
.ENABLE_AVI_PACKET(0),
.VBLANK_ONLY(1)
```

Teda:

```text
DATA_PREAMBLE + DATA_GB_LEAD + DATA_GB_TRAIL
bez DATA_PAYLOAD
```

### Interpretácia

```text
T2 PASS:
  data guard bandy sú na reálnom monitore pravdepodobne OK.
  Pokračuj T3.

T2 FAIL:
  problém je takmer určite v DATA_GB_LEAD / DATA_GB_TRAIL symboloch,
  ich kanálovom priradení alebo bitovej konvencii.
```

---

## Potom T3

Ak T2 prejde, nastav:

```systemverilog
.DEBUG_ISLAND_PHASES(3)
```

To je:

```text
DATA_PREAMBLE + guard bands + 1 payload symbol
```

### Interpretácia

```text
T2 PASS, T3 FAIL:
  problém je na začiatku TERC4 payloadu,
  formatter advance,
  bit-order TERC4,
  alebo mux zarovnanie.

T3 PASS, normal full packet FAIL:
  problém je až v 32-symbol payload dĺžke,
  ECC/payload obsahu,
  alebo trailing boundary po dlhšom payloade.
```

## Verdikt

Refaktor je dobrý a posúva debug správnym smerom. Aktuálny `soc_top.sv` však testuje normálny full GCP packet, ktorý už vieme, že zlyháva.

Ďalší potrebný bitstream je:

```text
DEBUG_ISLAND_PHASES=2
ENABLE_GCP_PACKET=1
ENABLE_AVI_PACKET=0
ENABLE_DATA_ISLAND=1
ENABLE_AUDIO=0
VBLANK_ONLY=1
```

Výsledok T2 nám povie, či je problém v guard band fáze alebo až v payload/TERC4 fáze.
