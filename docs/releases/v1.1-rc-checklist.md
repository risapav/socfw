# v1.1 Board and Native Pipeline Release Candidate Checklist

## Native Pipeline

- [x] Native pipeline is the default build path (no legacy backend required)
- [x] Legacy backend retained but not in default build
- [x] `RtlEmitter`, `SdcEmitter`, `FilesTclEmitter`, `BoardTclEmitter` emit directly without templates for HAL
- [x] `soc_top.sv` generated natively from `RtlModuleIR`

## AC608 Support

- [x] AC608 EP4CE15E22C8 board pack in `packs/builtin/boards/ac608_ep4ce15/board.yaml`
- [x] AC608 blink project golden anchor: `tests/golden/fixtures/ac608_blink/`
- [x] `clk` top port (AC608 board clock convention)
- [x] `ONB_LEDS[4:0]` — correct 5-bit LED vector
- [x] Device assignment: `EP4CE15E22C8`
- [x] LED pins: L3, J13, G16, B16, F8

## Board Selector Index

- [x] `socfw/board/selector_index.py` — `BoardSelectorIndex` dataclass with resources/aliases/profiles/connectors
- [x] Connectors are listed separately (displayable but not bindable)
- [x] `reports/board_selectors.json` emitted during build

## Feature Profiles and Selection

- [x] Board YAML supports `profiles:` with named resource sets
- [x] `features.profile: minimal` expands via `ProfileResolver`
- [x] `features.use: [board:@leds]` resolves aliases via `AliasResolver`
- [x] `expand_features()` produces flat selected resource path list
- [x] Auto-inference of selected features from bind targets when `features.use` absent

## Board Pin Conflict Detection

- [x] `BoardPinConflictRule` uses `PinUse` model for precise conflict messages
- [x] PIN001 fires only for resources active in the project (feature-aware)
- [x] Sub-path pruning avoids false positives (`external.sdram` + `external.sdram.dq`)
- [x] Error includes bit indices: `external.sdram.dq[11] and external.headers.P8.gpio[0]`

## Board.tcl Selected-Only Emission

- [x] `BoardTclEmitter` emits only selected/bound resources (not all onboard)
- [x] System clock/reset always emitted
- [x] IO standard assignments emitted per signal
- [x] External resources from bind targets also emitted
