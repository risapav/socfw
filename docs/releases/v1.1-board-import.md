# v1.1 Board Import Milestone

## Overview

This release adds comprehensive board modeling, validation, and import tooling.
The main new capability is the ability to describe FPGA boards in a structured
YAML format and use that description in project validation and RTL generation.

## What's new

### AC608 EP4CE15E22C8 board support

A canonical board descriptor for the AC608 Cyclone IV E board is included in
`packs/builtin/boards/ac608_ep4ce15/board.yaml`.

Includes:
- Onboard: LEDs, buttons, UART, I2C, HDMI (TMDS differential pairs)
- External: SDRAM (16-bit DDR-SDRAM with full control signals)
- Connectors: 3× GPIO header (P8/P5/P6) via `derived_resources`

### PMOD and connector-derived resources

Boards can now define physical connectors and derive logical resources from them:

```yaml
resources:
  connectors:
    pmod:
      J10:
        pins: [H1, F1, E1, C1, H2, F2, D2, C2]

derived_resources:
  - name: external.pmod.j10_led8
    from: connectors.pmod.J10
    role: led8
    top_name: PMOD_J10_LED
```

Supported roles: `gpio8`, `led8`, `button8`, `gpio14`, `gpio2`

### Board resource aliases and profiles

Boards can define short aliases and named feature profiles:

```yaml
aliases:
  leds: onboard.leds
  sdram: external.sdram

profiles:
  minimal:
    use: [onboard.leds]
  sdram:
    use: [external.sdram]
```

### Feature-aware pin conflict checking

Pin conflicts are only reported when both conflicting resources are actually
used in the same project (via `features.use` or port bindings).

### Width adaptation in port bindings

Port bindings now support `adapt: zero_extend`, `adapt: truncate`, and
`adapt: replicate` to bridge width mismatches between IP ports and board resources:

```yaml
ONB_LEDS:
  target: board:external.pmod.j10_led8
  adapt: zero_extend
```

### Board normalization pipeline

The board loader now runs a normalization pass before Pydantic validation:
- Infers missing `kind` from signal structure
- Converts legacy dict-style pins to canonical list format
- Expands `derived_resources` into bindable resources
- Emits BRD_ALIAS001–004 warnings for deprecated patterns

### `socfw board-info` command

```bash
socfw board-info ac608_ep4ce15
```

Shows board metadata, FPGA part, clock/reset, and all resources with pins.

### Board selector syntax documentation

See `docs/schema/board_selectors.md` for the canonical `board:` target syntax.

## Migration notes

- Boards using dict-style pins (`pins: {0: A1, 1: A2}`) will continue to work
  but will emit `BRD_ALIAS001` warnings. Migrate to list format: `pins: [A1, A2]`
- `soc_top_name` is deprecated in favor of `top_name`
- `dir` is deprecated in favor of `direction`
- `standard` is deprecated in favor of `io_standard`
